# 2026-01-08: Java 접근 제어자(Access Modifier) 학습

- 📚 **강의**: 김영한의 자바 기본 섹션 7

오늘은 객체 지향 프로그래밍의 핵심인 **캡슐화(Encapsulation)**를 완성하는 **접근 제어자**에 대해 학습했다. 외부 접근을 제한하여 데이터를 보호하고, 사용 편의성을 높이는 방법을 이해했다.

## 1. 접근 제어자 종류

자바는 4가지 종류의 접근 제어자를 제공하며, 접근 허용 범위가 좁은 순서대로 나열하면 다음과 같다.

1.  **private**: 모든 외부 호출을 막는다. (자기 자신의 클래스 내부에서만 접근 가능)
2.  **default** (package-private): 같은 패키지 안에서의 호출만 허용한다. (접근 제어자를 생략하면 적용됨)
3.  **protected**: 같은 패키지 안에서의 호출은 허용하며, 패키지가 달라도 상속 관계라면 호출을 허용한다.
4.  **public**: 모든 외부 호출을 허용한다.

## 2. 캡슐화 (Encapsulation)

캡슐화는 **데이터와 기능을 하나로 묶고**, 외부에 **꼭 필요한 기능만 노출**하는 것이다. 이를 위해 접근 제어자가 필수적이다.

### 데이터는 숨겨라 (`private`)
객체의 내부 데이터(멤버 변수, 필드)는 외부에서 함부로 조작할 수 없도록 `private`으로 막아야 한다. 데이터 변경은 반드시 객체가 제공하는 메서드를 통해서만 이루어져야 안전하다.

```java
public class Speaker {
    private int volume; // 외부 직접 접근 불가

    public void volumeUp() { // 메서드를 통해서만 조작 가능
        if (volume >= 100) {
            // ... 유효성 검사 로직
        } else {
            volume += 10;
        }
    }
}
```

### 기능은 꼭 필요한 것만 노출해라
내부적으로만 사용하는 복잡한 로직이나 보조 메서드는 `private`으로 숨기고, 사용자가 실제로 호출해야 하는 메서드만 `public`으로 열어두어야 한다. 이렇게 하면 사용자는 복잡한 내부 동작을 몰라도 편하게 기능을 사용할 수 있다.

## 3. 결론 (Best Practice)
- 기본적으로 모든 변수와 메서드는 외부에서 접근하지 못하도록 **최대한 숨기는 것**(`private`)이 좋다.
- 외부에서 사용해야 하는 기능만 선별적으로 `public`으로 노출한다.
