# 2026-01-12: Java 상속(Inheritance) 학습

- 📚 **강의**: 김영한의 자바 기본 섹션 10

오늘은 객체 지향 프로그래밍의 핵심 요소인 **상속**에 대해 학습했다. 부모 클래스의 기능을 자식 클래스가 물려받아 재사용하고 확장하는 방법을 익혔다.

## 1. 상속과 메모리 구조

```java
public class ElectricCar extends Car { ... }
```

- **extends**: 상속을 받을 때 사용하는 키워드다. (확장한다는 의미)
- **메모리 구조**: 자식 클래스(`ElectricCar`)의 인스턴스를 생성하면, 메모리 내부에는 **자식 클래스뿐만 아니라 부모 클래스(`Car`)의 공간도 함께 생성된다.**
    - 외부에서는 하나의 객체처럼 보이지만, 내부에는 부모와 자식이 공존한다.
- **호출 순서**: 메서드를 호출하면 **호출한 변수의 타입(클래스)**에서 먼저 기능을 찾고, 없으면 부모 클래스로 올라가서 찾는다.

## 2. 메서드 오버라이딩 (Overriding)

부모에게 상속받은 기능을 자식 클래스에서 **재정의(덮어쓰기)** 하는 것이다.

- **@Override**: 컴파일러가 오버라이딩 규칙(이름, 매개변수, 반환 타입 일치 여부)을 검사해준다. 실수를 방지하기 위해 붙이는 것이 권장된다.
- **조건**: 메서드 이름, 파라미터, 반환 타입이 같아야 한다. 접근 제어자는 부모보다 좁게 설정할 수 없다.

## 3. 상속과 접근 제어 (protected)

- **protected**: 같은 패키지 내에서는 접근을 허용하고, 패키지가 달라도 **상속 관계**라면 접근을 허용하는 접근 제어자다.
- 상속 관계에서 부모의 기능을 사용해야 할 때 유용하다.

## 4. super 키워드

### super 참조
부모 클래스의 필드나 메서드를 참조할 때 사용한다.
- `this`: 자기 자신의 멤버를 가리킴 (이름이 겹칠 때 사용)
- `super`: 부모의 멤버를 가리킴 (자식에서 재정의된 메서드 대신 부모의 원본 메서드를 호출할 때 등)

### super() 생성자
자식 클래스의 생성자에서는 **반드시 부모 클래스의 생성자를 호출해야 한다.**
- **규칙**: 생성자 첫 줄에 `super(...)`를 작성해야 한다.
- **생략 가능**: 부모에게 기본 생성자(매개변수 없음)가 있다면 생략 가능하며, 컴파일러가 자동으로 `super()`를 넣어준다.
- **필수 호출**: 부모에게 기본 생성자가 없고 매개변수가 있는 생성자만 있다면, 개발자가 직접 `super(...)`를 호출해줘야 한다.

## 5. 실습 예제 (계층 구조)

공통 속성(`name`, `price`)은 부모 클래스인 `Item`에 두고, 개별 속성(`artist`, `director`, `isbn` 등)은 각각의 자식 클래스(`Album`, `Movie`, `Book`)에 구현하여 코드 중복을 줄이고 구조를 개선했다.
