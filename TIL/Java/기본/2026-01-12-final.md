# 2026-01-12: Java final 키워드 학습

- 📚 **강의**: 김영한의 자바 기본 섹션 10

오늘은 변수의 값을 더 이상 변경하지 못하게 막아주는 **final** 키워드에 대해 학습했다. 지역 변수, 멤버 변수(필드), 그리고 상수에 적용될 때의 동작 방식을 이해했다.

## 1. final 지역 변수

지역 변수에 `final`을 붙이면 **최초 한 번만 할당**할 수 있다. 그 이후에는 값을 변경하려고 하면 컴파일 오류(`Cannot assign a value to final variable`)가 발생한다.

```java
final int data1;
data1 = 10; // 최초 할당 가능
// data1 = 20; // 변경 불가, 컴파일 오류
```

## 2. final 필드 (멤버 변수)

필드에 `final`을 사용할 경우, 반드시 **초기화**를 해줘야 한다. 초기화 방법은 두 가지가 있다.

1.  **생성자 초기화**: 생성자를 통해 값을 받아 할당한다. (인스턴스마다 다른 값을 가질 수 있음)
2.  **필드 초기화**: 선언과 동시에 값을 할당한다. (모든 인스턴스가 같은 값을 가짐)
    - 필드에서 이미 값을 할당했다면, 생성자에서 다시 값을 할당할 수 없다.

## 3. static final (상수)

"변하지 않고 공용으로 사용되는 값"을 **상수(Constant)**라고 한다. 자바에서는 `static final` 키워드를 사용하여 상수를 정의한다.

- **static**: 메모리를 효율적으로 사용하기 위해 공용 영역(메서드 영역)에 저장하고 공유한다.
- **final**: 값을 변경할 수 없도록 막는다.

### 관례
상수는 대문자로만 작성하며, 단어와 단어 사이는 언더스코어(`_`)로 구분한다. (UPPER_SNAKE_CASE)

```java
public static final double PI = 3.14159;
public static final int MAX_USERS = 1000;
```

## 4. final 참조형 변수

참조형 변수(객체를 가리키는 변수)에 `final`을 붙이면 **참조값(Address)을 변경할 수 없다.** 즉, 다른 객체로 갈아탈 수 없다.

**하지만, 참조하는 대상(객체)의 내부 값(필드)은 변경할 수 있다.**
`final`은 "변수 자체"의 값을 못 바꾸게 하는 것이지, "변수가 가리키는 객체"의 내부 상태까지 막는 것은 아니기 때문이다.

```java
final Data data = new Data();
// data = new Data(); // 참조값 변경 불가 (컴파일 오류)

data.value = 10; // 참조 대상의 내부 값은 변경 가능
data.value = 20; // 변경 가능
```
