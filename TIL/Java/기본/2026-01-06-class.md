# 2026-01-06: Java 클래스(Class) 학습

- 📚 **강의**: 김영한의 자바 기본 섹션 2

오늘은 자바 객체 지향 프로그래밍의 기초가 되는 **클래스(Class)**에 대해 학습했다. 클래스가 왜 필요한지, 그리고 객체가 메모리에서 어떻게 동작하는지를 중점적으로 이해했다.

## 1. 클래스가 필요한 이유

학생의 이름, 나이, 성적을 관리한다고 할 때, 변수를 각각 선언해서 관리하면 학생이 늘어날 때마다 변수가 기하급수적으로 늘어나 관리하기 어렵다. 또한 이름, 나이, 성적은 서로 연관된 데이터임에도 불구하고 따로 떨어져 있어 데이터의 응집도가 떨어진다.

클래스를 사용하면 서로 관련된 데이터를 하나의 개념(타입)으로 묶어서 효율적으로 관리할 수 있다.

## 2. 클래스와 객체

### 클래스 (Class)
- 객체를 생성하기 위한 **설계도** 또는 틀이다.
- 개발자가 직접 정의하는 새로운 데이터 타입(사용자 정의 타입)이다.
- 붕어빵 틀에 비유할 수 있다.

### 객체 (Object) / 인스턴스 (Instance)
- 클래스(설계도)를 기반으로 실제 메모리에 만들어진 **실체**다.
- 붕어빵 틀에서 구워져 나온 붕어빵에 비유할 수 있다.
- **인스턴스**: 특정 클래스로부터 생성된 객체임을 강조할 때 사용하는 용어다. (예: student1은 Student 클래스의 인스턴스다.)

## 3. 객체 생성과 참조

클래스를 기반으로 객체를 생성하려면 `new` 키워드를 사용해야 한다.

```java
Student student1 = new Student(); // 객체 생성
```

### 동작 원리
1.  **메모리 할당**: `new Student()`를 실행하면 메모리(Heap 영역)에 `Student` 객체가 생성된다.
2.  **참조값 반환**: 객체가 생성되면 자바는 그 객체에 접근할 수 있는 **참조값(메모리 주소)**을 반환한다.
3.  **참조값 저장**: 반환된 참조값이 변수(`student1`)에 저장된다.

**중요**: 변수에는 실제 객체가 들어있는 것이 아니라, **객체의 위치를 가리키는 참조값**이 들어있다.

## 4. 객체와 배열

객체도 배열에 담아서 관리할 수 있다. 이때 배열에는 객체 자체가 저장되는 것이 아니라, **객체의 참조값**이 저장된다.

```java
Student[] students = new Student[2];
students[0] = student1; // 참조값 복사
students[1] = student2; // 참조값 복사
```

## 5. 주의사항 (심화)

> **"자바에서 대입은 항상 변수에 들어있는 값을 복사해서 전달한다."**

객체가 들어있는 변수를 다른 변수에 대입할 때도 이 원칙은 적용된다. 단, 객체 자체가 복사되는 것이 아니라 **참조값만 복사**되어 전달된다는 점을 명심해야 한다. 따라서 두 변수는 같은 객체(같은 메모리 주소)를 가리키게 된다.
