# 2026-01-07: Java 객체 지향 프로그래밍 (OOP) 학습

- 📚 **강의**: 김영한의 자바 기본 섹션 4

오늘은 절차 지향 프로그래밍에서 **객체 지향 프로그래밍(OOP)**으로 넘어가는 과정과 그 필요성에 대해 학습했다. 음악 플레이어(MusicPlayer) 예제를 발전시키며 두 방식의 차이를 명확하게 이해할 수 있었다.

## 1. 절차 지향 vs 객체 지향

### 절차 지향 프로그래밍 (Procedural Programming)
- **실행 순서(절차)**를 중요하게 생각하는 방식이다.
- 프로그램의 흐름을 순차적으로 처리하는 것에 초점을 둔다.
- **데이터(Data)**와 해당 데이터를 처리하는 **기능(Function/Method)**이 분리되어 있다.

### 객체 지향 프로그래밍 (Object-Oriented Programming)
- **객체(Object)**를 중요하게 생각하는 방식이다.
- 실제 사물의 속성과 기능을 분석하여 프로그래밍에 반영한다.
- **데이터**와 **기능**을 **하나로 묶어서(클래스)** 관리한다.

## 2. 음악 플레이어 예제 (Evolution)

### 단계 1: 절차 지향 - 변수 사용
데이터(`volume`, `isOn`)가 여러 변수에 흩어져 있고, 기능은 `main` 메서드 등에 절차적으로 나열되어 있다. 데이터 관리가 어렵다.

### 단계 2: 절차 지향 - 데이터 묶음 (클래스 도입)
`MusicPlayerData` 클래스를 만들어 데이터를 묶었지만, 여전히 데이터와 기능(메서드)은 분리되어 있다. 기능을 수행할 때마다 데이터를 인수로 전달해야 한다.

```java
// 데이터는 별도 클래스로 존재
public class MusicPlayerData {
    int volume;
    boolean isOn;
}

// 기능은 외부에 정의 (데이터를 인수로 받아야 함)
static void volumeUp(MusicPlayerData data) {
    data.volume++;
}
```

### 단계 3: 객체 지향 - 캡슐화 완성
`MusicPlayer` 클래스 내부에 **데이터**(`volume`, `isOn`)와 **기능**(`on()`, `off()`, `volumeUp()`)을 함께 정의했다.

```java
public class MusicPlayer {
    int volume; // 데이터
    boolean isOn;

    void volumeUp() { // 기능 (자신의 데이터를 사용)
        volume++;
    }
}
```

- **장점**:
    - **자율성**: 객체가 자신의 데이터를 스스로 관리한다. 외부에서는 객체의 메서드만 호출하면 된다.
    - **단순함**: 사용하는 입장(`main` 메서드)에서는 객체의 내부 데이터 구조를 알 필요가 없다.

```java
// 사용하는 입장 (Main)
MusicPlayer player = new MusicPlayer();
player.volumeUp(); // 단순하고 직관적
```

## 3. 캡슐화 (Encapsulation)

> **"속성과 기능을 하나로 묶어서 필요한 기능을 메서드를 통해 외부에 제공하는 것"**

객체 지향의 가장 중요한 특징 중 하나다. 관련된 데이터와 로직을 하나의 캡슐(클래스) 안에 가두어 보호하고, 외부에는 꼭 필요한 기능만 노출한다.

- **높은 응집도**: 관련된 코드가 한곳에 모여 있어 이해하기 쉽다.
- **유지보수 용이성**: 내부 로직이 변경되어도 외부 사용하는 코드에는 영향을 주지 않는다.
