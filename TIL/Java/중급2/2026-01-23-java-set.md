# 2026-01-23: Java Set 구현체 (Set)

- 📚 **강의**: 김영한의 자바 중급2 섹션 9 Set

오늘은 자바가 제공하는 다양한 `Set` 구현체들의 차이점과, 집합 연산(합집합, 교집합, 차집합)을 편리하게 수행하는 방법에 대해 학습했다.

## 1. Java Set 구현체 비교

자바는 `Set` 인터페이스를 구현한 3가지 주요 클래스를 제공한다. 상황에 맞춰 선택해야 한다.

| 구현체 | 특징 | 입력 순서 보장 | 정렬 | 성능 |
| :--- | :--- | :--- | :--- | :--- |
| **HashSet** | 해시 알고리즘 사용. 가장 일반적. | ❌ | ❌ | **O(1)** (가장 빠름) |
| **LinkedHashSet** | 해시 + 연결 리스트. 입력한 순서대로 저장됨. | ✅ | ❌ | **O(1)** (HashSet보다 약간 느림) |
| **TreeSet** | 이진 탐색 트리(Red-Black Tree) 사용. 값에 따라 정렬됨. | ❌ | ✅ | **O(log N)** (데이터 정렬이 필요할 때) |

```java
Set<String> hashSet = new HashSet<>();       // 순서 없음 
Set<String> linkedSet = new LinkedHashSet<>(); // 입력 순서 보장
Set<String> treeSet = new TreeSet<>();       // 가나다(123) 순 정렬
```

## 2. 집합 연산 (Set Operations)

자바의 `Set`은 수학의 집합 기능을 메서드로 제공한다.

```java
Set<Integer> set1 = new HashSet<>(List.of(1, 2, 3));
Set<Integer> set2 = new HashSet<>(List.of(3, 4, 5));

// 합집합 (Union)
set1.addAll(set2); // set1 = [1, 2, 3, 4, 5]

// 교집합 (Intersection)
set1.retainAll(set2); // set1 = [3] (공통된 요소만 남김)

// 차집합 (Difference)
set1.removeAll(set2); // set1 = [1, 2] (set2에 있는 요소 제거)
```

## 3. 리스트의 중복 제거

리스트에 중복된 데이터가 있을 때, `Set`을 이용하면 아주 쉽게 중복을 제거할 수 있다.

```java
List<Integer> list = List.of(1, 2, 2, 3, 3);
Set<Integer> uniqueSet = new HashSet<>(list); // 생성자에 컬렉션을 넣으면 자동 중복 제거!
```
