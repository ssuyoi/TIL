# 2026-01-20: Java 배열 리스트 (ArrayList)

- 📚 **강의**: 김영한의 자바 중급2 섹션 3

오늘은 자바에서 가장 많이 사용되는 자료구조 중 하나인 **ArrayList**의 내부 구조와 성능 특징에 대해 학습했다.

## 1. 배열의 특징과 한계

`ArrayList`를 이해하려면 먼저 기본 배열(`Array`)의 특징을 알아야 한다.
- **장점 (Fast Access)**: 인덱스를 통해 데이터에 **O(1)**로 접근할 수 있다. (메모리 주소 직접 계산 가능)
- **단점 (Fixed Size)**:
    - 생성할 때 크기를 지정해야 하며, 이후 변경할 수 없다.
    - 데이터를 추가하려고 하는데 꽉 찼다면, 더 큰 배열을 만들고 데이터를 복사(Copy)해야 하는 비용이 든다.
    - 중간에 데이터를 추가하거나 삭제할 때, 뒤의 데이터들을 한 칸씩 밀거나 당기는 **이동(Shift) 연산**이 필요하다.

## 2. ArrayList 구현 (MyArrayList)

`ArrayList`는 배열의 불편함을 해소하기 위해 **동적 크기 조절** 기능을 추가한 클래스다.

### 핵심 기능
- **동적 사이즈 (Grow)**: 데이터가 꽉 차면 자동으로 배열의 크기를 2배(또는 1.5배)로 늘리고, 기존 데이터를 복사한다.
- **제네릭 지원**: `<E>`를 사용하여 타입 안전성을 보장한다.

### 성능 (Big-O)

| 연산 | 시간 복잡도 | 설명 |
| :--- | :--- | :--- |
| **인덱스 조회/변경** (`get`, `set`) | **O(1)** | 배열이므로 인덱스로 즉시 접근 가능 (매우 빠름) |
| **마지막에 추가** (`add`) | **O(1)** | 보통 O(1)이지만, 배열이 꽉 차서 늘려야 할 때는 O(N) (Amortized O(1)) |
| **중간 추가/삭제** (`add(index, ..)`) | **O(N)** | 해당 인덱스 뒤의 모든 요소를 이동(Shift)시켜야 하므로 느림 |
| **순차 검색** (`indexOf`) | **O(N)** | 데이터를 하나씩 확인해야 함 |

> **💡 결론**: 데이터를 순차적으로 추가하고 조회하는 경우에 가장 적합하다. 중간 삽입/삭제가 빈번하다면 성능이 떨어질 수 있다.
