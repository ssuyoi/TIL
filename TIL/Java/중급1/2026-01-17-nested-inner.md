# 2026-01-17: Java 중첩 클래스와 내부 클래스 (Nested & Inner Class)

- 📚 **강의**: 김영한의 자바 중급1 섹션 8

오늘은 클래스 안에 다른 클래스를 정의하는 **중첩 클래스**에 대해 학습했다. 중첩 클래스는 크게 **정적 중첩 클래스**와 **내부 클래스**로 나뉜다.

## 1. 중첩 클래스 분류

| 종류 | 선언 위치 | 키워드 | 설명 |
| :--- | :--- | :--- | :--- |
| **정적 중첩 클래스** (Static Nested) | 클래스 내부 | `static class` | 바깥 클래스의 인스턴스와 무관함. 단순히 위치만 내부일 뿐. |
| **내부 클래스** (Inner) | 클래스 내부 | `class` | 바깥 클래스의 **인스턴스**에 소속됨. 바깥 인스턴스의 참조를 가짐. |
| **지역 클래스** (Local) | 메서드 내부 | `class` | 메서드 내부에서만 사용 가능. |
| **익명 클래스** (Anonymous) | 메서드 내부 | `new 클래스() {}` | 이름이 없음. 선언과 생성을 동시에 함. |

## 2. 정적 중첩 클래스 vs 내부 클래스

가장 중요한 차이는 **"바깥 클래스의 인스턴스에 접근할 수 있는가?"**이다.

### 정적 중첩 클래스 (`static class`)
```java
static class Nested { ... }
```
- 바깥 클래스의 `static` 멤버에는 접근 가능하지만, **인스턴스 멤버(`instanceValue`)에는 접근할 수 없다.**
- 바깥 클래스 인스턴스 없이도 생성 가능하다. (`new Outer.Nested()`)
- 사실상 다른 클래스인데, 패키지 관리상 묶어둔 것에 가깝다.

### 내부 클래스 (`class`)
```java
class Inner { ... }
```
- 바깥 클래스의 **모든 멤버(private 포함)에 접근 가능하다.**
- 바깥 클래스의 인스턴스가 있어야만 생성 가능하다. (`outer.new Inner()`)
- **메모리 오버헤드**: 내부 클래스의 인스턴스는 생성될 때 자동으로 바깥 클래스 인스턴스의 참조를 보관한다. (메모리를 더 사용하므로 신중히 사용해야 함)

## 3. 사용 이유

1.  **논리적 그룹화**: 특정 클래스가 다른 하나의 클래스 안에서만 사용될 때, 이를 내부에 두어 관련성을 명확히 한다.
2.  **캡슐화 강화**: 바깥 클래스의 `private` 멤버에 접근할 수 있으므로, 긴밀하게 연결된 작업을 수행하면서도 외부에는 감출 수 있다.

> **💡 결론**: 바깥 클래스의 인스턴스 멤버에 접근해야 한다면 **내부 클래스**를, 그렇지 않다면 **정적 중첩 클래스**를 사용하자. 불필요한 메모리 낭비를 막기 위해 가능하면 `static`을 붙이는 것이 좋다.
