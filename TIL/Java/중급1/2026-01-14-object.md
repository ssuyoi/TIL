# 2026-01-14: Java Object 클래스 학습

- 📚 **강의**: 김영한의 자바 중급1 섹션 2

오늘은 자바의 최상위 부모 클래스인 **Object** 클래스에 대해 학습했다. 모든 클래스가 묵시적으로 상속받는 이 클래스가 제공하는 공통 기능(`toString`, `equals`)과 다형성 지원 메커니즘을 이해했다.

## 1. java.lang 패키지

자바의 가장 기본이 되는 클래스(`Object`, `String`, `System` 등)들을 모아둔 패키지다.
- **특징**: `import` 없이 자동으로 사용할 수 있다.

## 2. Object 클래스

자바에서 모든 클래스의 **최상위 부모 클래스**다.
- 클래스에 상속받는 부모가 없으면, 컴파일러가 자동으로 `extends Object`를 추가한다.
- **목적**:
    1.  **공통 기능 제공**: 모든 객체에게 필요한 기본 기능(`toString`, `equals`, `getClass` 등)을 제공한다.
    2.  **다형성 지원**: `Object`는 모든 객체를 참조할 수 있다.

## 3. Object와 다형성

`Object`는 모든 클래스의 부모이므로, **모든 객체(인스턴스)를 참조할 수 있다.**

```java
Object dog = new Dog(); // 가능
Object car = new Car(); // 가능
```

- **장점**: 모든 객체를 대상으로 하는 범용적인 기능(예: `action(Object obj)`)이나 배열(`Object[]`)을 만들 수 있다.
- **한계**: `Object` 타입으로 참조하면 `Object`가 가진 기능만 호출할 수 있다. 자식의 고유 기능(`sound()`, `move()`)을 사용하려면 **다운캐스팅**이 필요하다.

## 4. toString()

객체의 **문자열 표현**을 반환하는 메서드다.
- **기본 반환값**: `클래스명@해시코드(16진수)` (예: `dog.Dog@a09ee92`)
- **System.out.println()**: 내부적으로 객체의 `toString()`을 호출하여 출력한다.

### 오버라이딩 (Overriding)
기본 반환값은 객체의 상태(데이터)를 파악하기 어렵다. 따라서 유용한 정보(필드 값 등)를 제공하도록 **재정의**해서 사용한다.

```java
@Override
public String toString() {
    return "Dog{name='" + name + "', age=" + age + "}";
}
```

## 5. equals()

객체의 **같음**을 비교하는 메서드다.

- **동일성 (Identity, `==`)**: 두 객체의 **참조값(메모리 주소)**이 같은지 비교한다. (물리적으로 같은 객체인지)
- **동등성 (Equality, `equals`)**: 두 객체의 **내부 데이터(논리적 값)**가 같은지 비교한다.

### 기본 동작 및 오버라이딩
- `Object`의 기본 `equals()`는 **`==` 비교(동일성 비교)**를 수행한다.
- 따라서 객체의 내용(예: `id`, `name`)이 같을 때 "같다"고 판단하려면, 반드시 **`equals()`를 오버라이딩해서 동등성 비교 로직을 구현해야 한다.**

```java
@Override
public boolean equals(Object o) {
    if (this == o) return true; // 참조값 같으면 통과
    if (o == null || getClass() != o.getClass()) return false; // 타입 체크
    User user = (User) o;
    return Objects.equals(id, user.id); // 내부 값 비교
}
```
