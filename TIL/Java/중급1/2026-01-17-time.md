# 2026-01-17: Java 날짜와 시간 (Date & Time)

- 📚 **강의**: 김영한의 자바 중급1 섹션 7

오늘은 Java 8부터 도입된 **Java Time API (JSR-310)**에 대해 학습했다. 과거의 `Date`, `Calendar` 클래스의 문제점을 해결한 현대적인 날짜/시간 라이브러리다.

## 1. Java Time API 등장 배경

### 기존 방식(Date, Calendar)의 문제점
- **가변 객체(Mutable)**: 날짜 객체의 값을 변경하면 원본이 바뀐다. 이는 공유 참조 시 심각한 사이드 이펙트를 유발한다.
- **설계 결함**: 월(Month)이 0부터 시작하는 등 직관적이지 않다.

### Java Time API 특징
- **불변 객체(Immutable)**: 모든 날짜/시간 객체는 불변이다. 값을 변경하면 항상 새로운 객체를 반환한다.
- **명확한 분리**: 날짜(`LocalDate`), 시간(`LocalTime`), 둘 다(`LocalDateTime`), 타임존 포함(`ZonedDateTime`) 등으로 개념이 명확히 분리되어 있다.

## 2. 주요 클래스

- **`LocalDate`**: 날짜만 표현 (년, 월, 일). (예: `2024-01-01`)
- **`LocalTime`**: 시간만 표현 (시, 분, 초). (예: `12:30:45`)
- **`LocalDateTime`**: 날짜와 시간을 모두 표현.
- **`ZonedDateTime`**: `LocalDateTime` + **타임존(TimeZone)** 정보 포함. (글로벌 서비스에 필수)

## 3. 기간과 시간 간격 (Period vs Duration)

시간의 간격을 표현하는 두 가지 클래스가 있다.

- **`Period`**: **날짜(Date)** 단위의 간격. (년, 월, 일)
    - 예: "이 프로젝트는 3개월 남았다." (`Period.ofMonths(3)`)
    - `getYears()`, `getMonths()`, `getDays()`

- **`Duration`**: **시간(Time)** 단위의 간격. (시, 분, 초, 나노초)
    - 예: "라면 끓이는 시간은 3분이다." (`Duration.ofMinutes(3)`)
    - `toHours()`, `toMinutes()`, `getSeconds()`

## 4. 주요 유틸리티

- **생성**: `now()` (현재), `of(...)` (지정 값)
- **계산**: `plusDays()`, `minusMinutes()`, `withYear()` (변경)
    - **주의**: 불변 객체이므로 반환값을 꼭 받아야 한다. (`date = date.plusDays(1)`)
- **비교**:
    - `isBefore()`: 이전인가?
    - `isAfter()`: 이후인가?
    - `isEqual()`: 같은가? (타임존이 달라도 시간적으로 같으면 true)
