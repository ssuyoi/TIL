# 2026-01-03 ~ 2026-01-05: Java 배열(Array) 학습

- 📚 **강의**: 김영한의 자바 입문 섹션 9

배열의 기본 개념부터 2차원 배열, 향상된 for문까지 학습했다. 마지막으로 상품 관리 프로그램을 직접 만들어보고 강사님의 코드와 비교하며 부족한 점을 점검했다.

## 1. 배열 (Array)

같은 타입의 변수를 사용하기 편하게 하나로 묶어둔 것이다.
- **선언**: `int[] students;` (아직 메모리에 생성되지 않음)
- **생성**: `students = new int[5];` (메모리에 5개의 int 공간 생성, 참조값 반환)
- **인덱스**: `0`부터 시작하며, `n-1`까지 존재한다.

### 리펙토링
배열을 사용하면 변수 이름을 하나하나 관리할 필요 없이 `index`를 사용하여 데이터를 효율적으로 관리할 수 있다. 반복문과 결합하면 그 효과가 극대화된다.

## 2. 2차원 배열 (2D Array)

행(row)과 열(column)로 구성된 배열이다.
- **초기화**:
  ```java
  int[][] arr = {
      {1, 2, 3},
      {4, 5, 6}
  };
  ```
- **순회**: 중첩 `for` 문을 사용하여 탐색한다. `arr.length`는 행의 개수, `arr[row].length`는 해당 행의 열의 개수다.

## 3. 향상된 for문 (Enhanced For)

배열의 처음부터 끝까지 순회할 때 유용하다 ("for-each" 문이라고도 함). 인덱스를 직접 관리하지 않아 코드가 간결하고 가독성이 좋다.

```java
for (int number : numbers) {
    System.out.println(number);
}
```
단, 인덱스 값이 필요할 때는 일반 `for` 문을 사용해야 한다.

## 4. 실습 및 코드 비교 (`ArrayEx7` vs `ArrayEx7_1`)

`상품 관리 프로그램`을 작성한 후 강사님의 코드와 비교해보았다.

### 4.1. 구조 개선 (`if` vs `else if`)
- **나의 코드**: 모든 메뉴 선택에 독립적인 `if` 문을 사용했다.
  ```java
  if (choice == 1) { ... }
  if (choice == 2) { ... } // 1번이 실행되 2번도 불필요하게 검사함
  ```
- **강사님 코드**: `if - else if - else` 구조를 사용하여 불필요한 조건 검사를 줄이고, 잘못된 메뉴 선택(`else`)까지 처리가 가능하다.

### 4.2. 유지보수성 (상수 활용)
- **나의 코드**: 배열의 크기나 조건에 숫자 `10`을 직접 적었다 (매직 넘버).
- **강사님 코드**: `maxProductCount` 변수를 사용하여 최대 상품 개수를 한 곳에서 관리할 수 있게 했다. 나중에 개수를 변경할 때 변수 값만 바꾸면 되므로 유지보수에 유리하다.

### 4.3. 입력 처리 (버퍼 비우기)
- **나의 코드**: `next()`를 사용하여 공백 전까지만 이름을 입력받았고, `nextInt()` 후 줄바꿈 처리에 대한 고려가 부족했다.
- **강사님 코드**: `nextInt()` 후 `nextLine()`을 호출하여 입력 버퍼에 남은 엔터(`\n`)를 제거했다. 이를 통해 이후 문자열 입력 시 발생할 수 있는 오류를 방지하고, 상품 이름에 공백이 포함되어도(예: "제주 감귤") `nextLine()`으로 온전히 입력받을 수 있다.

```java
int choice = sc.nextInt();
sc.nextLine(); // 입력 버퍼를 비운다.
```

### 4.4. 예외 처리
- **강사님 코드**: 메뉴에 없는 번호를 입력했을 때 "잘못된 메뉴를 선택하셨습니다."라고 알려주는 `else` 블록이 있어 사용자 경험(UX)이 더 좋다.
