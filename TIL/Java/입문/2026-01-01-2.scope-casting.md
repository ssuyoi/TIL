# 2026-01-01: Java 스코프(Scope)와 형변환(Casting) 학습

- 📚 **강의**: 김영한의 자바 입문 섹션 6

오늘은 변수의 접근 범위인 **스코프(Scope)**와 데이터 타입을 변환하는 **형변환(Casting)**에 대해 학습했다.

## 1. 스코프 (Scope)

변수에 접근할 수 있는 범위를 말한다. 일반적으로 `{}` 블록 내부에 선언된 변수는 그 블록 안에서만 접근 가능하다.

### 스코프의 존재 이유
1.  **메모리 효율성**: 변수가 필요한 구간에서만 메모리를 차지하고, 블록이 끝나면 제거되므로 메모리를 아낄 수 있다.
2.  **코드 복잡성 감소**: 변수의 사용 범위를 제한하면, 코드를 분석할 때 생각해야 할 범위가 줄어들어 유지보수에 좋다.

### for문과 스코프
`while` 문은 반복 변수(`i`)를 반복문 밖에서 선언해야 하지만, `for` 문은 초기화 식에서 선언하므로 스코프를 루프 내부로 제한할 수 있어 더 깔끔하고 안전하다.

```java
// 좋은 예 (for문)
for (int i = 0; i < 10; i++) { ... } // i는 for문 밖에서 접근 불가

// 아쉬운 예 (while문)
int i = 0;
while (i < 10) { ... } // i가 while문 종료 후에도 살아있음
```

## 2. 형변환 (Casting)

데이터 타입을 변경하는 것을 말한다.

### 자동 형변환 (Implicit Casting)
**작은 범위**에서 **큰 범위**로 대입할 때는 자바가 자동으로 형변환을 해준다. 데이터 손실이 없기 때문이다.
- `int < long < double`
```java
int intValue = 10;
double doubleValue = intValue; // 자동 형변환 (10 -> 10.0)
```

### 명시적 형변환 (Explicit Casting)
**큰 범위**에서 **작은 범위**로 대입할 때는 데이터 손실이 발생할 수 있으므로, 개발자가 직접 `(type)`을 적어주어야 한다.
```java
double doubleValue = 1.5;
int intValue = (int) doubleValue; // 1 (소수점 버림)
```

### 오버플로우 (Overflow)
형변환 시 표현 범위를 초과하는 값을 대입하면 전혀 다른 숫자가 되어버리는 현상이다.
```java
long maxIntOver = 2147483648L; // int 최대값 + 1
int intValue = (int) maxIntOver; // -2147483648 (오버플로우 발생)
```

### 계산과 형변환
- 같은 타입끼리의 계산은 같은 타입의 결과를 낸다 (`int / int = int`).
- 서로 다른 타입의 계산은 더 큰 범위로 자동 형변환되어 계산된다.
```java
int a = 3;
int b = 2;
double result = (double) a / b; // 3.0 / 2 -> 3.0 / 2.0 -> 1.5
```
