# 2026-01-05: Java 메서드(Method) 학습

- 📚 **강의**: 김영한의 자바 입문 섹션 10

오늘은 중복되는 코드를 효율적으로 관리하고 모듈화할 수 있는 **메서드(Method)**에 대해 학습했다. 특히 자바의 가장 중요한 대원칙인 **Call by Value**에 대해 깊이 있게 이해하는 시간을 가졌다.

## 1. 메서드 (Method)

특정 작업을 수행하는 코드 블록이다. 수학의 함수와 유사하게 입력을 받아 처리를 수행하고 결과를 반환한다.

### 메서드 정의
```java
public static int add(int a, int b) { // 메서드 선언
    return a + b; // 메서드 본문
}
```

### 용어 정리
- **매개변수 (Parameter)**: 메서드를 정의할 때 선언하는 변수. 메서드 내부에서 사용할 값을 전달받는다. (예: `int a`, `int b`)
- **인수 (Argument, 인자)**: 메서드를 호출할 때 실제로 전달하는 값. (예: `add(1, 2)`에서 `1`, `2`)

### 메서드 사용의 장점
1.  **중복 제거**: 같은 코드를 여러 번 작성하지 않고 메서드를 호출하여 재사용할 수 있다.
2.  **수정 용이**: 로직을 수정해야 할 때 메서드 내부만 변경하면 모드 곳에 적용된다.
3.  **가독성**: 메서드 이름으로 코드의 기능을 명확하게 표현할 수 있다.

## 2. 메서드 오버로딩 (Overloading)

**이름이 같아도 매개변수가 다르면** 다른 메서드로 정의할 수 있는 기능이다.

### 오버로딩 규칙
- **가능**: 매개변수의 타입, 개수, 순서가 다르면 가능하다.
- **불가능**: 반환 타입만 다른 경우에는 오버로딩할 수 없다.

### 메서드 시그니처 (Method Signature)
자바가 메서드를 구분하는 기준이다. **메서드 이름**과 **매개변수 타입(순서)**로 구성된다. 반환 타입은 시그니처에 포함되지 않기 때문에 반환 타입만 다르면 컴파일 오류가 발생한다.

## 3. 자바의 대원칙 (Pass by Value)

> **"자바는 항상 변수의 값을 복사해서 대입한다."**

이 원칙은 기본형(Primitive)뿐만 아니라 참조형(Reference)에도 동일하게 적용된다.

### 동작 원리
메서드를 호출할 때 전달하는 인수의 **값(Value)**이 복사되어 매개변수에 전달된다. 따라서 메서드 내부에서 매개변수의 값을 변경해도 호출자의 변수에는 영향을 주지 않는다 (기본형의 경우).

```java
int num1 = 5;
changeNumber(num1); // num1의 값(5)을 복사해서 전달
System.out.println(num1); // 여전히 5
```

이 원칙을 이해하는 것이 자바의 메모리 구조와 객체 지향 프로그래밍을 이해하는 핵심이다.

## 4. 반환 타입 (Return Type)

- **반환 값이 있는 경우**: 반드시 `return` 키워드를 사용하여 값을 반환해야 한다.
- **`void`**: 반환 값이 없는 경우 사용하며, `return`을 생략할 수 있다. (중간에 종료하고 싶을 때 `return;` 사용 가능)
